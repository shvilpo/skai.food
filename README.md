# skai.food

Личный дневник питания (PWA): калории, белок, масса растительной пищи и клетчатка за день; сводка за неделю.

Без бэкенда: все данные хранятся в IndexedDB на устройстве, бэкап — экспорт/импорт JSON в настройках.

## Запуск локально

```bash
python3 -m http.server 8123
# открыть http://localhost:8123
```

## Деплой на GitHub Pages

1. Создать публичный репозиторий, запушить содержимое этой папки в ветку `main`.
2. В настройках репозитория: Settings → Pages → Source: Deploy from a branch → `main`, папка `/ (root)`.
3. Через минуту приложение доступно по `https://<username>.github.io/<repo>/`.

Все пути в проекте относительные, поэтому подпапка GitHub Pages работает без настройки.

## Установка на iPhone

1. Открыть адрес приложения в Safari.
2. Поделиться → «На экран „Домой"» (Add to Home Screen).
3. Запускать с иконки — приложение работает в полноэкранном режиме и офлайн (кроме будущих функций распознавания).

Обновления подхватываются автоматически: при открытии приложение показывает кэш и в фоне подтягивает свежую версию (видна со следующего открытия).

## Структура

- `index.html`, `css/app.css` — оболочка и стили (светлая/тёмная тема)
- `js/app.js` — вкладки: Дневник, Продукты, Неделя, Настройки
- `js/db.js` — IndexedDB (products / entries / settings)
- `js/seed.js` — стартовая база из ~30 базовых продуктов (Скурихин/USDA)
- `sw.js`, `manifest.webmanifest`, `icons/` — PWA

## Модель данных

- Продукт: КБЖУ-поля на 100 г (ккал, белок, клетчатка) + «растительная доля, %» для расчёта растительной массы. Рафинированные продукты (масло, сахар) растительной массой не считаются.
- Запись дневника хранит снапшот значений продукта, поэтому правка продукта не меняет прошлые записи.
